# 股票估值汇总页设计（Bull/Base/Bear + 实时当前价）

- 日期：2026-06-04
- 状态：设计已确认，待写实现计划

## 目标

提供一个汇总页面，一行一只股，同时展示每只股的 **Bear / Base / Bull 三档每股内在价值**、**实时当前价** 与 **安全边际**，并支持页面上「刷新」按钮按需拉取实时价格。

## 关键决策（brainstorming 结论）

1. **数据来源**：独立数据文件，与文档 frontmatter 解耦（不改 `_docs_schema.py`）。
2. **产出形态**：Flask 页面（HTML 表格），非静态 markdown。
3. **不做独立 skill**：改为页面上「刷新」按钮，点击自动拉实时价。
4. **初始填充**：Agent 一次性从存量 buffett 文档抽取三档估值。
5. **价格拉取机制**：方案 A —— 缓存加载 + AJAX 刷新按钮（契合现有 `supply_chain.py` 的「页面 + /api」模式）。

## 现状背景

- 三档估值目前**只写在各 buffett 文档正文散文**中（如「内在价值 ≈ 189 亿（7.78 元/股）」「bull 情景 215 亿 / 8.87 元」），frontmatter 无结构化估值字段。
- 现有 Flask 结构：blueprint 在 `app/routes/__init__.py` 定义、`app/__init__.py` 注册、配置 dict 放 `app/config/`、模板 `app/templates/`、API 路由用 `jsonify`（参考 `app/routes/supply_chain.py`）。
- 所有股票数据统一走 `UnifiedStockDataService`（两层缓存 + force_refresh 语义）。

## 1. 数据文件（唯一源）

`docs/stock-analytics/valuations.yaml`，git 跟踪。每只股一条：

```yaml
- stock_code: '000878'
  stock_name: 云南铜业
  market: A            # A / US / HK
  currency: CNY        # CNY / USD / HKD
  rating: watch
  bear: 6.50           # 每股内在价值（原币：元/股 或 USD/HKD）
  base: 7.78
  bull: 8.87
  conviction_date: '2026-06-02'
  source_doc: sectors/materials/nonferrous/2026-06-02-云南铜业-buffett分析.md
  note: 三情景加权 base 7.78   # 可选
```

约定：
- 三档统一为**每股原币值**。文档若只给总市值 + 股本，抽取时换算成每股。
- 缺某档 → 该字段留空/`null`，页面显示「—」。
- 同一 `stock_code` 多篇文档 → 按 `conviction_date` 取最新一篇，`source_doc` 记其相对 `docs/stock-analytics/` 的路径。
- `stock_code` 必须字符串引号（防 YAML int 化丢前导 0），与 docs frontmatter 约定一致。

## 2. 初始填充（Agent 一次性抽取）

一次性脚本（**不入库**，跑完 `rm`，符合 dev-conventions「一次性脚本不入库」）：

- Glob `docs/stock-analytics/sectors/**/*buffett分析.md` + `docs/stock-analytics/cross-sector/**/*buffett*.md`（后者一档含 2 只股，各抽一条）。
- 逐篇读结论摘要/估值段，抽出 bear/base/bull 每股值、currency（按 market 推 CNY/USD/HKD）、rating（读 frontmatter）、conviction_date、source_doc。
- 同 `stock_code` 多篇取 `conviction_date` 最新。
- 写出 `docs/stock-analytics/valuations.yaml`（`encoding='utf-8'`）。

此后 `valuations.yaml` 为权威源，手工维护。**已知取舍**：文档更新后需手动同步 yaml，二者可能短期脱节。

## 3. Flask 路由层

- `app/routes/__init__.py` 增加 `valuations_bp`（url_prefix `/valuations`）；`app/__init__.py` 注册。
- 新建 `app/routes/valuations.py`：
  - `GET /valuations/`：`load_valuations()` 读 yaml → 取**缓存**当前价（`get_realtime_prices(codes)`，不 force_refresh）→ 算各档安全边际 → `render_template('valuations.html', rows=...)`。
  - `GET /valuations/api/prices?force=1`：`get_realtime_prices(all_codes, force_refresh=True)` **批量一次**拉（不逐 code 循环）→ 返回 `{code: {current_price, name, margin_bear, margin_base, margin_bull}}`。
- `load_valuations()` helper 放路由模块：`yaml.safe_load` 读固定路径，返回 list[dict]。

## 4. 安全边际计算

对每档值 V 与当前价 P：`margin = V / P - 1`（正 = 上行空间，负 = 高估）。

- 比例量纲与币种无关 → A/US/HK 混表无需汇率换算。
- P 为 `None`/`0` → 当前价与各 margin 显示「—」/「N/A」（price 可能为 None/0，按 data-fetch-conventions 防御）。
- `get_realtime_prices` 名称回退（yfinance 返回 name=code）→ 名称一律用 yaml `stock_name`。
- 字段命名按实际：`current_price`（非 price）。

## 5. 前端模板

`app/templates/valuations.html` extends `base.html`；`base.html` 导航加入口。

- 表格列：`代码 | 名称 | 板块 | 评级 | Bear | Base | Bull | 当前价 | Base安全边际 | conviction日期 | 文档链接`
- 默认按 Base安全边际降序（最低估在上）。
- Base安全边际：正绿负红；评级 badge 配色。
- 右上「🔄 刷新」按钮 → fetch `/valuations/api/prices?force=1` → 原地更新当前价/各档 margin 单元格 + 显示「更新时间」。
- 刷新失败 → 保留缓存值，按钮旁提示「刷新失败」。

## 6. 错误/边界处理

- yaml 缺失/空 → 空表 + 提示文案。
- 某股无价/退市 → 行仍在，价显示「—」。
- 缺某档估值 → 「—」。
- 批量拉价整体失败 → 保留缓存值，前端提示「刷新失败」。

## 7. 测试

- **纯函数单测**（无需 app）：`load_valuations()` 解析；`compute_margin(V, P)` 覆盖 P=None / P=0 / 缺档 / 正常。
- **路由 smoke**：`create_app()`（`SCHEDULER_ENABLED=0`）下 `GET /valuations/` 返回 200（HTML 渲染需走 create_app，因 base.html 跨 blueprint url_for 会抛 BuildError）。
- **API 测试**：monkeypatch `get_realtime_prices` 避免真实联网，断言返回结构与 margin 计算。

## 非目标（YAGNI）

- 不改 `_docs_schema.py` / frontmatter。
- 不做独立 skill。
- 不做汇率换算（margin 为比例，币种无关）。
- 不做服务端价格快照文件（UnifiedStockDataService 已有两层缓存）。
- 不做自动「文档→yaml」同步（手工维护）。
