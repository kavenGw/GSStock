# 盯盘页相对走势图 — 设计

- **日期**：2026-07-01
- **范围**：盯盘页（watch.html）价值洼地区块新增一张「相对走势」归一叠加折线图，与价值洼地表共用 7天/30天/90天 toggle；A+H 双重上市标的同时画 A、H 两条线。

## 目标

在盯盘页价值洼地区块内，价值洼地表上方新增一张多股相对走势折线图：所有盯盘股在所选周期起点归一到 0%，叠加对比累计涨跌幅，一眼看谁跑赢跑输。对真·A+H 双重上市标的，A 腿与 H 腿各画一条独立归一的线做对比。

## 现状（复用点）

- 价值洼地区块已在 `app/templates/watch.html` 底部，含 `#period-toggle`（7d/30d/90d，默认 30d）+ 价值洼地表（`#stock-body`）。
- `app/static/js/value_dip.js`：`DOMContentLoaded` 一次 `fetch('/value-dip/api/stocks')` 加载，toggle 切换只客户端重渲染表（`change_7d/30d/90d` 一次全返回）。
- `app/routes/value_dip.py` + `app/services/value_dip.py`：`ValueDipService.get_watch_performance()` 用 `WatchService.get_watch_list()`（读 `WATCH_CODES`）取 codes，`get_trend_data(codes, days=90)` 取 90 天 OHLC，算涨幅/高点回退。
- 盯盘池唯一权威源 `app/config/stock_codes.py` 的 `WATCH_CODES`（每条 `{code, name, market}`）。
- watch 页已引入 ECharts。

## 关键决策

1. **图表语义 = 归一叠加折线**：所有股票在所选周期起点归一到 0%，画多条累计涨跌幅折线叠在同一张图。（非 vs 基准超额、非单期涨幅排行。）
2. **数据流 = 服务端按周期归一（方案 A）**：切 7/30/90 时前端向服务端请求该周期已归一的序列，服务端计算归一后返回。
3. **A+H 来源 = WATCH_CODES 加 `ah` 字段**：在需要双显的盯盘条目上加可选 `ah` 子字段承载对应腿代码，与盯盘池同源维护，零猜测。
4. **范围 = 盯盘池全部**：图覆盖 `WATCH_CODES` 全部股票（价值洼地表那一批），含 A/HK/US/KR。仅真·A+H 双重上市标的额外画对应腿。

## 架构

### 数据模型

`WATCH_CODES` 条目加可选 `ah` 字段，仅真·A+H 双重上市标的加：

```python
{'code': '2631.HK', 'name': '天岳先进', 'market': 'HK',
 'ah': {'code': '688234', 'market': 'A', 'name': '天岳先进'}}
```

- 已知 A+H：天岳先进 H `2631.HK` ↔ A `688234`；江西铜业 H `0358.HK` ↔ A `600362`。
- 其余盯盘股是否真 A+H 需**实现时逐一核实**（akshare A+H 列表 / 人工核对），确认后才加 `ah`。不硬猜（如 `9981.HK 沃尔核材` 是否真有 H 股需验证）。
- 非 A+H（小米/腾讯/泡泡玛特=港股或 ADR+HK 可互转、SK海力士=韩股）不加 `ah`，只画一条线。

### 服务层

`ValueDipService.get_relative_series(period: str) -> list`（`period ∈ {'7d','30d','90d'}`）：

1. 从 `WATCH_CODES` 收集 watch codes + 所有 `ah` 腿 code，合并去重后一次 `get_trend_data(all_codes, days=90)`（命中缓存后重复调用零成本；与价值洼地表同一份 90 天数据源）。
2. 按 period 映射天数（`{'7d':7,'30d':30,'90d':90}`），**按交易日计数**截取每条序列最近 N 个交易日窗口（与 `value_dip.py` 现有 `idx = len(data)-days` 口径一致）。
3. 每条线归一到窗口起点：`value = (close/base − 1) × 100`，`base` = 窗口首个有效收盘。
4. 展开 A+H：带 `ah` 的盯盘条目产出两条 series（主腿 + ah 腿），label 分别加 `(H)`/`(A)` 后缀（按各腿 market 标注）。

返回：

```json
{"series": [
  {"code": "2631.HK", "name": "天岳先进", "market": "HK",
   "label": "天岳先进(H)", "dates": ["2026-04-01", ...], "values": [0.0, 1.23, ...]}
]}
```

`dates` 与 `values` 等长；`values` 已归一为累计 %。

### 路由

`GET /value-dip/api/relative?period=7d|30d|90d`（默认 30d）→ `ValueDipService.get_relative_series(period)` → `jsonify({'series': ...})`；整体 `try/except` 失败 `{'error': ...}, 500`（与现有 `/value-dip/api/*` 一致）。价值洼地表用的 `/value-dip/api/stocks` 保持不变。

### 前端

`app/templates/watch.html`：价值洼地区块内、`#period-toggle` 下方、价值洼地表上方，新增一个 ECharts 容器 `<div id="relative-chart">`。

`app/static/js/value_dip.js`：
- `DOMContentLoaded`：按默认 30d 拉一次相对走势 + 初始化 ECharts 实例。
- toggle handler 扩展：切周期时**既**客户端重渲染表**又** `fetch('/value-dip/api/relative?period='+currentPeriod)` 重取并 `setOption` 重绘图。
- 图形态：多条归一折线（~14–15 条），x 轴交易日期 + `connectNulls`（兜跨市场休市错位），y 轴 %，0% 基准线；A+H 两腿 label 带 (A)/(H)。
- 交互：legend 点击隐藏/显示单线；tooltip 悬停某日期显示各线累计涨幅并按值排序。

## 错误处理

- 端点整体 `try/except` → 500。
- 单股序列缺失/取数失败 → 跳过该 series，其余照常（图不整挂）。
- `ah` 腿取数失败 → 只丢那条腿，主腿保留。
- 前端 fetch 失败 → `console.error` + 图区显「加载失败」，不影响表。

## 测试

`tests/test_value_dip_relative.py`（平铺，轻量 Flask 注入 `Flask()+register_blueprint(value_dip_bp)`，不走 create_app）：

- `get_relative_series` 归一正确性：窗口起点=0%、`(close/base−1)×100`。
- period 截窗按交易日计数：7d 取最近 7 个交易日。
- A+H 展开：带 `ah` 的条目产出两条 series、label 带 (A)/(H)、各含等长 dates/values。
- 降级：单股缺失序列 / `ah` 腿失败 → 跳过不抛。
- 端点 200 + 结构契约：`series` 列表、每条 `dates`/`values` 等长。
