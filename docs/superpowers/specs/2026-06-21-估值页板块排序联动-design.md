# 估值页板块（组）随列排序联动 — 设计

- 日期：2026-06-21
- 涉及页面：`/valuations`（Claude估值 / 价值洼地）
- 关联文件：`app/templates/valuations.html`（主）、`app/routes/valuations.py`（仅参考，不改）
- 相关规则：`.claude/rules/portfolio-valuations.md`（估值汇总页一节）

## 目标

分组模式下，点击 **Bear / Base / Bull** 任一安全边际列时，不仅组内行重排，**板块（组）之间的先后也按该列联动重排**，使最被低估的板块浮到顶部。点击 **评级 / 日期** 列时，组顺序保持默认（按标的数量降序），只排组内行。

## 范围

**纯前端改动**：`app/templates/valuations.html` 内联 JS + 必要的模板属性。

- **不改** `app/routes/valuations.py`：服务端已渲染每行 `data-mbear / data-mbase / data-mbull` 以及 group-header（`data-sector`），并已按「数量降序」输出组顺序，前端有足够信息重排，无需新接口或新字段。
- 不改 `valuations.yaml`、不改 `group_by_sector` 的服务端分组逻辑。

## 现状（改动前）

- 列排序（评级/Bear/Base/Bull/日期）由 JS `sortBy()` → `applySort()` 实现。
- `applySort()` 在分组模式下**只重排组内行**，组与组之间的先后固定为服务端 `group_by_sector` 的输出（`groups.sort(key=lambda g: (-g['count'], g['sector']))`，即按标的数量降序、并列按 sector key）。
- 前端无法改变组顺序；`switchMarket` 只调 `applyMarketFilter`（隐藏不匹配行 + 隐藏空组），不触发排序。

## 核心规则

1. **板块代表值**：某组按当前排序列排序后「**首行**」对应的边际值。
   - 降序时 = 组内最大值（最低估个股）；升序时 = 组内最小值。与排序方向自洽。
   - 代表值取「**首个通过当前市场过滤的行**」的值，使组顺序与可见内容一致。
2. **联动触发列**：仅 `bear` / `base` / `bull`。`rating` / `date` 不触发组重排。
3. **默认组顺序**：排序列为 `rating` / `date`（或任何非联动态）时，组顺序回退到服务端原始顺序（按数量降序）。
4. **空值处理**：代表值为 `null` 的组（该组在当前市场过滤下无有效边际行）沉到底部，与现有「null 行恒末位」规则一致。
5. **平铺模式**：无分组，组重排不适用，现有 `mode === 'flat'` 分支不变。

## 实现要点

### 1. 记住原始组顺序
init（`initValuations`）时从服务端已渲染的 group-header DOM 顺序捕获：

```js
const defaultGroupOrder = [...document.querySelectorAll('#val-table .group-header')]
  .map(h => h.dataset.sector);
```

服务端渲染顺序即「按数量降序」，作为非联动态的回退顺序。

### 2. 改造 `applySort()`（grouped 分支）
- 先对每组组内行排序（沿用现有 `sortRows`）。
- 计算目标组顺序：
  - 若 `['bear','base','bull'].includes(sortKey)`：对每个 sector 取「首个市场匹配行」的 `marginOf(row, sortKey)` 作代表值，按 `sortDir`（null 末位）排序 sector 列表。
  - 否则：用 `defaultGroupOrder`。
- 调 `reorderGroups(orderedSectors)` 把组整块搬移到位。

### 3. 新增 `reorderGroups(orderedSectors)`
按目标 sector 顺序，依次把每组的 header + 其所有 `tr.grp-<sector>[data-code]` 行 `appendChild` 回 tbody（整块搬移、保持组内已排好的行序）。未出现在 `orderedSectors` 中的组（理论上不会有）保底追加在末尾。

### 4. 市场过滤联动
`switchMarket` 末尾在 `applyMarketFilter()` 之后增加 `applySort()`，使切换 A/港/美股时组顺序按可见行的代表值重算。

> 注意调用顺序：代表值依赖「市场匹配」判断，需保证市场态（`currentMarket` 与各行 `data-market-hidden`）已更新。`applyMarketFilter()` 先跑、再 `applySort()`，或在 `applySort` 内直接用 `currentMarket` 与 `tr.dataset.market` 判断匹配（不依赖 `applyMarketFilter` 写的 `marketHidden`），二选一，实现时取后者更稳（解耦先后依赖）。

### 5. 刷新价联动
`refreshPrices()` 已在更新 dataset 后调用 `applySort()`，组顺序随新边际自动重排，无需改动。

## 首屏效果

默认排序为 `base desc`，页面加载后 `initValuations → setMode → applySort` 即把组按「各板块最优 Base 安全边际」排列，取代旧的「按数量」首屏顺序——更契合「价值洼地」页用途。首屏会先以服务端数量序渲染、再由 JS 瞬时重排（无感知闪烁）。

## 验证（手动）

分组模式下：
- 点 Base / Bear / Bull → 组顺序变化且与排序方向自洽（降序时含最低估个股的组在顶）。
- 反复点同列切换升/降 → 组顺序随方向翻转。
- 点评级 / 日期 → 组回到数量序，仅组内行变。
- 切换市场（A/港/美股）→ 组按当前市场可见行的代表值重排；空组隐藏（现有行为）。
- 刷新实时价 → 组随新边际重排。

平铺模式：不分组，组重排不生效，行按列全局排（现有行为不变）。

边界：
- 含 null 边际的组在联动态沉底。
- carve-out 组（如「啤酒」）与 `__none__`（未分类）参与同一套规则。
- 某 sector key 含中文 / `__none__`：选择器沿用现有 `CSS.escape`。

## 非目标（YAGNI）

- 不增加独立的「板块排序」控件（按板块名/数量手动切）。
- 不持久化组顺序（由 `sortKey`/`sortDir` 派生，已随 `PREF_KEY` 持久）。
- 不改服务端分组或排序逻辑。
