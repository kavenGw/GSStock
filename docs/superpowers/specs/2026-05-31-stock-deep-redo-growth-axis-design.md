# stock-deep-redo 成长轴优化设计

> 状态：已批准，待写实施计划
> 日期：2026-05-31
> 前置：`2026-05-31-stock-deep-redo-sector-lens-design.md`（sector-lenses 架构）

## 背景与问题

`stock-deep-redo` skill 现有 4 个 lens（AI 横切 / PCB-CCL / 存储 DRAM-NAND / 存储 NOR）重心偏
**供给侧 + 涨价弹性 + 估值纪律**。对 AI 基建 / 存储 / PCB 这类**成长股**，只看估值会系统性踩两个坑：

1. **采证缺口**：客户自身的出货/capex/营收增长预期没作硬性必查；标的自身扩产（capex/达产爬坡）在
   存储/AI lens 里几乎缺位。
2. **估值缺口**：playbook §3 场景加权是**周期均值归一化**导向（"拒绝用周期顶利润定价"）。对真正的
   结构性成长股，纯均值回归会**系统性低估**——bull 情景存在但缺一个"增长持续性"的证据底座来支撑
   它的概率和倍数。

核心诉求（用户原话）：AI 相关 / 存储 / PCB 标的，**不能只看估值，还要看增长、扩产情况、主要客户的
增长预期**。

## 设计张力（必须守住的灵魂）

skill 现有灵魂是**反炒作**：拒绝周期顶利润定价、诚实面对"贵"、未兑现概念不许进估值。加"看增长"
若做歪会退化成追涨工具。因此增长轴必须**证据接地**（具名客户的 capex/出货 guidance 带 URL），
而非叙事拔高。解法是一条**对偶约束**：既防周期顶高估，也防缺证据时被增长叙事拔高；但反过来，
增长证据全硬的真成长股，允许 base 适度脱离纯周期均值，避免系统性低估。

## 已定决策

| 维度 | 选择 |
|------|------|
| 挂载方式 | 新增横切 lens + 改 playbook 估值机制 |
| 估值机制 | 增长证据包门控 bull（三情景保留，证据不足封顶 bull 概率） |
| 客户粒度 | 分层兑证：具名大客户优先，保密/分散时降级终端市场总量兑底 |
| 触发范围 | 默认对每只股跑识别（与 AI lens 对称） |

## 改动清单（3 文件）

### 改动 1 — `references/sector-lenses.md`：新增横切 lens

头部说明：从"AI 是唯一横切 lens"改为"**AI 与 成长 是两个横切 lens，默认对每只股都跑识别**"。

在 AI 节之后追加一节「**成长 / 扩产 / 客户增长（横切 · 默认对每只股跑识别）**」，沿用五段式：

- **【识别信号】**：判标的是否在产能扩张 / 营收高增轨道（区别于纯周期/价值股）。强信号：在建产线/扩产
  capex 公告、近年营收 CAGR 显著、AI 基建/存储/PCB 高增赛道、管理层把增长归因到扩产达产或大客户放量。
  判为纯周期/价值股（无扩产、纯靠涨价弹性）→ 一句话带过，不强跑全清单。
- **【必查清单（采证 face）】**：
  - **标的自身扩产**：在建/规划产线、capex 金额与节奏、投产时间表、达产率爬坡曲线、对应增量营收/产能
    （量化，带公告 URL + 日期）。
  - **客户增长预期（分层兑证）**：① 先具名——识别 Top-N 大客户，逐个找其自身 capex/出货/营收 guidance
    （如沪电→英伟达/北美服务器 ODM capex 指引）；② 客户保密/过于分散 → 降级到终端市场总量（北美 CSP
    capex、AI 服务器出货预测、AMOLED 面板出货等第三方机构口径），明写这是终端兑底而非具名；
    ③ 每条标【硬】（客户/机构公开 guidance）/【软】（媒体推测）/【缺】。
  - **历史增长质量**：近 3 年营收/净利 CAGR、增长拆量价、是否伴随毛利改善（有质量增长 vs 烧钱扩张）。
  - **跑道长度（TAM 渗透）**：标的当前在赛道的渗透率/份额，距天花板还有多远。
- **【撰写落点（撰写 face）】**：
  - §2 市场规模：补"跑道长度"（标的在 TAM 中的当前渗透与剩余空间）。
  - §3 盈利能力：增长拆量价 + 增长是否伴随毛利改善（增长质量）。
  - §6 核心新论点：扩产达产逻辑（结构性增量 vs 周期性补涨二分）。
  - §9 估值：成长持续性证据包（扩产达产确定性 + 客户 capex 能见度 + TAM 跑道）作为 bull 情景赋权依据。
- **【双面必答】**：
  - 扩产是结构性需求支撑，还是赌行业景气（同业一起扩 → 未来产能过剩/价格战）？最强反驳前置。
  - 客户增长预期是已锁定订单/capex 指引，还是框架意向/卖方一致预期（容易证伪）？
  - 高增长是否已被股价 price-in（高 PE/PB）——增长再好也要算安全边际，不许用"成长空间"稀释"贵"。
- **【监控指标模板】**：
  - 在建产能达产率 miss 进度 / 稼动率连续 N 季下滑（扩产证伪）。
  - 大客户 capex/出货指引下修，或终端市场（AI 服务器/CSP capex/AMOLED）增速预测下调。
  - 营收增速连续 N 季放缓且毛利率同步走弱（增长质量恶化）。
  - 同业扩产产能集中释放窗口临近（供给过剩前瞻信号）。

### 改动 2 — `references/playbook.md`：§3 场景加权加「增长证据包门控 bull」

§3 铁律追加一条对偶约束（措辞落地）：

> **bull 情景增长证据包门控**：bull 的概率与倍数上修必须由「成长持续性证据包」三要素支撑——
> (a) 扩产达产确定性（产线已开工/设备到位/达产路线清晰=硬；仅规划=软）、(b) 客户 capex/出货能见度
> （具名客户公开 guidance=硬；终端总量=中；卖方一致预期=软）、(c) TAM 跑道（渗透率低、空间大）。
> 三项里硬证据越少，bull 概率越要封顶（如三项全软 → bull 概率 ≤ 20%）。这是"拒绝周期顶定价"的
> **对偶约束**：既不许用周期顶利润定价（防高估），也不许在缺乏增长证据时给 bull 高权重（防被叙事
> 拔高）。**反过来**，三项全硬的结构性成长股，base 情景也可适度脱离纯周期均值（用穿越周期的成长
> 中枢），避免系统性低估真成长。

§2（13 节说明）轻量补：§2 加"跑道长度"、§3 盈利能力加"增长拆量价 + 增长质量"、§9 估值开头用证据包
作为 bull 赋权依据。**不新增节号，13 节结构不变**（成长内容并入 §2/§3/§6/§8/§9）——保持现有审查
清单的节数校验不破。

§4（AI 维度标签法 → sector-lenses 指针）旁边补一句指向新成长 lens 的指针，避免漏读。
