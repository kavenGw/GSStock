# 预警合并进盯盘页（仅 WATCH_CODES）设计

- 日期：2026-06-29
- 状态：已确认设计，待写实现计划

## 1. 背景与目标

当前有两个独立页面：

- `/alert` 预警中心：按分类/持仓覆盖全 A 股，前端 `signal-detector.js` 算 OHLC 技术信号 + 后端缓存信号；含分类筛选、信号类型筛选、阈值设置、详情弹窗、回测胜率。
- `/watch` 盯盘助手：固定池 `WATCH_CODES`（A/港/韩多市场），按市场分组渲染（每市场一张图 + 一张 summary 表），已含分时图、AI 分析、支撑阻力、九转、季报、基准标的栏、市场状态轮询。

目标：**删除独立 `/alert` 页，把技术信号能力收缩进盯盘页**，天然只覆盖 `WATCH_CODES`。在每市场 summary 表为每只盯盘股内联展示技术信号。

## 2. 范围

纳入：
- 全量移植预警信号到盯盘页 summary 表的新列「信号」：RSI 超买/超卖、MACD 金叉/死叉/背离、布林上下轨/收窄、成交量异动、均线突破，+ 买卖形态（缩量突破/突破新高/顶部巨量/突破跌破 5 日均线/双底/双顶）——即 `signal-detector.js` `detectAll` 的全部输出。
- 保留阈值设置（RSI 超买/超卖阈值、放量倍数），localStorage 持久化，盯盘页头部齿轮入口。
- 删除独立 `/alert` 页及其专属资源。

不纳入：
- 分类筛选、信号类型（买/卖/中性）筛选：池子只剩 WATCH_CODES，天然无用，舍弃。
- 回测胜率：舍弃（`/api/backtest/*` 随 alert.py 一并删）。
- 预警中心的财报日历倒计时（`/alert/api/earnings`）：盯盘页已有自己的季报展示，不移植。
- 九转、支撑阻力：盯盘页已有，不重复（九转徽标贴股票名后，支撑阻力在价格区间条）。

## 3. 架构与数据流

### 3.1 后端（新增）

`GET /watch/signals`（watch_bp）：
- 对 `WATCH_CODES` 批量调 `unified_stock_data_service.get_trend_data(codes, days=60)`。
- 返回 `{'success': True, 'stocks': [{'stock_code', 'data': [60日 OHLC]}, ...]}`（沿用 unified 服务返回结构，前端转 `{code: data}` 字典）。
- 一次批量调用覆盖全部盯盘股；多市场由 unified 服务内部处理。

### 3.2 前端（`watch.js`）

- 初始化阶段拉 `/watch/signals` → 存 `this.ohlc[code]`。
- 复用 `signal-detector.js` 的 `SignalDetector.detectAll(ohlc, thresholds)` 逐股计算 → 存 `this.signals[code]`。
- 渲染进 `_renderSummaryTable` 的新列「信号」。
- 阈值改动时**纯前端重算 + 重渲染**（无后端往返）——这是保留阈值设置的核心价值。
- 阈值从 localStorage（沿用 `WatchStore` 持久化机制）读取，默认值：RSI 超买 70 / 超卖 30 / 放量倍数 2.0。

### 3.3 `signal-detector.js`

保持不变、不删（heavy_metals 页也在用）。直接复用其 `detectAll(ohlcData, thresholds)`。

## 4. 前端呈现

### 4.1 summary 表列变化

当前：`股票 | 涨跌% | 支撑/现价/压力 | AI摘要`

变更后：`股票 | 涨跌% | 信号 | 支撑/现价/压力 | AI摘要`

- 「信号」列位于「涨跌%」右侧、「支撑/现价/压力」左侧（已确认）。
- 展示该股当前命中的信号徽标：买=绿、卖=红、中性=灰，多个并排；无信号显示 `—`。
- 复用盯盘页已有 `.entry-signal` 徽标样式与配色，与九转徽标视觉一致。

### 4.2 设置入口

- 盯盘页头部（page-header 区）加齿轮按钮。
- 弹窗复用预警阈值设置：RSI 超买阈值（slider 60–90）、RSI 超卖阈值（slider 10–40）、放量倍数（slider 1.5–5，step 0.5）。
- 保存 → 写 localStorage → 重算信号 → 重渲染所有 summary 表。

## 6. 测试

- 路由层（轻量，`Flask() + register_blueprint(watch_bp)` 注入，避免 create_app 副作用）：
  - `GET /watch/signals` 返回结构含 `stocks`，每项有 `stock_code` 与 `data`。
  - 删除后访问 `/alert` 返回 404（或确认 alert_bp 未注册）。
- 前端信号计算：复用未改动的 `signal-detector.js`，不新增前端测试（无既有惯例）。

## 7. 风险与注意

- `/watch/signals` 60 日 OHLC 批量取数：港股/美股冷缓存可能较慢；与 `/watch/prices` 的 `cache_only` 不同，本接口需真实 OHLC。可接受首屏信号列稍后填充（先渲染 `—`，OHLC 到位后补算），与现有 chart 异步填充模式一致。
- 信号列在非交易时段仍可计算（OHLC 历史数据始终可用），符合盯盘页 OHLC 走势「非交易时间仍可获取」的既有约定。
