# 盯盘助手优化设计

## 概述

三项优化：布局重构、九转信号、财报数据面板。

## 一、布局重构

### 现有 → 新布局

```
现有:                              新:
┌──────────────────────────┐      ┌──────────────────────────────────┐
│ 标题行                    │      │ 标题行              [九转徽章]   │
├──────────┬───────────────┤      ├──────────────────────────────────┤
│ 分时图    │  AI分析边栏   │      │ 分时图（100%宽）                  │
│ (70%)    │  (30%)        │      │   ┌─────────┐                    │
│          │               │      │   │九转:买7  │ ← 左下角浮动       │
├──────────┴───────────────┤      │   └─────────┘                    │
│ 分析标签页面板             │      ├────────────────┬─────────────────┤
│ (realtime / 7d / 30d)    │      │ AI分析          │ 财报数据         │
└──────────────────────────┘      │ [实时|7d|30d]  │ 近4季营收/利润   │
                                  │ 信号+摘要+S/R  │ 季末股价区间     │
                                  └────────────────┴─────────────────┘
```

### 变更要点

- 删除右侧 30% 的 `analysis-sidebar`
- 图表从 `col-md-8` 改为 `col-12`，高度保持 200px
- 下方分析区改为左右 `col-md-6` 双栏
- 左栏复用现有标签页逻辑（realtime/7d/30d），内容紧凑化
- 右栏新增财报面板
- 标题行右侧加九转状态徽章

## 二、九转信号（TD Sequential）

### 算法

- 买入计数：连续N天收盘价 < 4天前收盘价，计数1→9
- 卖出计数：连续N天收盘价 > 4天前收盘价，计数1→9
- 计数达到9触发信号，中断则重置
- 需要至少13天日K数据

### 数据流

```
UnifiedStockDataService.get_trend_data(code, days=30)
  → TDSequentialService.calculate(ohlcv_data)
  → { direction: 'buy'|'sell'|null, count: 0-9, completed: bool, history: [...] }
  → /watch/chart-data 接口附带返回 td_sequential 字段
  → 前端渲染
```

### 新增文件

- `app/services/td_sequential.py`

### 展示

- 标题行徽章：`买7`(绿) / `卖9✓`(红) / 无信号不显示
- 图表左下角浮动：ECharts graphic.text，半透明背景，`TD买入 7/9` 或 `TD卖出 9/9 ✓`
- 计数 ≥7 高亮提醒

## 三、财报数据

### 数据源

| 市场 | 数据源 | 方法 |
|------|--------|------|
| A股 | akshare | 利润表 + 营收 |
| 美股/港股 | yfinance | quarterly_financials |

### 季末股价

通过 `UnifiedStockDataService.get_trend_data()` 获取历史OHLC，提取每季度末（3/6/9/12月最后交易日）的收盘价、最高价、最低价，平均价=(最高+最低)/2

### 新增文件

- `app/services/earnings_service.py`

### 缓存

- 复用 `UnifiedStockCache`，cache_type=`earnings`，TTL=7天

### 展示
