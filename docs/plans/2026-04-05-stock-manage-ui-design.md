# 股票代码管理 UI 优化 — 增强行内编辑

## 目标

优化股票代码管理页面的布局和编辑交互体验，核心改善：表格列宽更合理、编辑控件更友好、操作反馈更明确。

## 范围

### 包含

1. 表格列宽重新分配
2. 标签列独立展示（pill 样式）
3. 编辑控件按字段类型差异化
4. 统一的保存/取消操作按钮
5. hover 可编辑提示

### 不包含

- 新增股票表单位置调整
- 批量操作功能扩展
- 分类层级展示方式变更
- 拖拽排序
- 删除操作变更

## 设计详情

### 1. 表格列宽调整

当前列宽分配不均，投资建议占据过多空间，代码和名称被压缩。

**优化后列宽：**

| 列 | 当前 | 优化后 |
|---|---|---|
| 代码 | 6% | 7% |
| 名称 | 8% | 9% |
| 别名 | 7% | 8% |
| 标签 | （无独立列） | 10% |
| 投资建议 | 剩余（~49%） | 剩余（~39%） |
| 分类 | 15% | 12% |
| 操作 | 80px | 70px |

### 2. 标签列独立

当前标签（tags）和投资建议共享空间，需拆分为独立列。

- 标签用 **pill 样式**（圆角背景色标签）展示
- 多个标签横向排列，自动换行
- 颜色采用低饱和暗色背景 + 亮色文字

### 3. 编辑控件增强

#### 3.1 hover 提示

- 可编辑单元格（`.editable`）hover 时背景微亮
- 显示铅笔小图标（CSS `::after` 伪元素），提示可双击编辑

#### 3.2 普通字段（代码 / 名称 / 别名）

- 双击后替换为单行 `<input>`
- input 右侧显示 **✓ 保存** 和 **✕ 取消** 按钮
- 快捷键：`Enter` 保存，`Escape` 取消
- 编辑区域加蓝色边框标识

#### 3.3 投资建议

- 双击后替换为 `<textarea>`（默认 4 行高，`resize: vertical` 可拖拽调整）
- textarea 下方右对齐显示 **保存** 和 **取消** 按钮
- 快捷键：`Ctrl+Enter` 保存（Enter 用于换行），`Escape` 取消
- 最小高度 80px

#### 3.4 标签

- 双击后进入标签编辑模式
- 已有标签显示为 pill + **×** 删除按钮
- 末尾跟一个 `<input>` 用于输入新标签
- `Enter` 添加标签（不保存），点击 **×** 删除标签
- 底部显示 **保存** 和 **取消** 按钮，点击保存时提交所有变更
- `Escape` 取消整个编辑

## 涉及文件

| 文件 | 变更 |
|---|---|
| `app/templates/stock_manage.html` | 列宽 CSS、编辑交互 JS（双击处理、控件渲染、保存/取消逻辑） |
| `app/templates/partials/stock_row.html` | 标签列独立、pill 渲染 HTML |

## 技术要点

- 纯前端变更，不涉及后端 API 或数据模型修改
- 编辑 JS 逻辑集中在 `stock_manage.html` 的 `<script>` 块中
- pill 样式用内联 CSS 或 `<style>` 块实现，无需新建 CSS 文件
- 保持现有 API 调用方式不变（`PUT /stocks/<code>`, `PUT /stocks/<code>/tags` 等）
