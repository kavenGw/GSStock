# 盯盘助手 UI 重设计

## 目标

重构盯盘助手页面，按市场分组显示股票，增加市场状态、刷新倒计时、通知状态和关键点位信息。

## 后端变更

### `/watch/prices` 接口扩展

在现有价格数据基础上，合并返回：

- `notification_threshold`：波动率阈值（从 watch_analysis 取，无分析时用默认值 0.02）
- `cooldown_remaining`：冷却剩余秒数（0=就绪），从策略实例的冷却记录计算
- `support_levels` / `resistance_levels`：从 watch_analysis 表取当日数据

### `/watch/market-status` 新接口

返回各活跃市场的状态信息：

```json
{
  "A": {"name": "A股", "timezone": "Asia/Shanghai", "time": "10:54", "status": "trading", "status_text": "交易中"},
  "US": {"name": "美股", "timezone": "America/New_York", "time": "21:54", "status": "closed", "status_text": "已收盘"}
}
```

通过 TradingCalendarService 服务端计算，避免客户端时区不准。

## 前端变更

### watch.js

- `renderCards()` → `renderGroups()`：按 market 字段分组，每组一个市场 Header
- 市场 Header：名称 + 当地时间（每秒更新） + 交易状态指示灯
- 股票卡片精简：移除单独的市场标签和交易状态，改为显示支撑/阻力位 + 通知状态
- 全局倒计时：页面顶部显示下次刷新秒数，每秒递减，归零触发刷新
- 无股票的市场不显示

### watch.html

- 调整页面结构适配分组布局
- 倒计时元素放在标题栏右侧

## 通知状态数据流

```
WatchAssistantStrategy._notification_cooldowns (内存dict)
    ↓
watch.py /prices 路由读取策略实例的冷却记录
    ↓
计算每只股票的 cooldown_remaining 秒数
    ↓
前端显示 "就绪" 或 "冷却中 剩Xm"
```
