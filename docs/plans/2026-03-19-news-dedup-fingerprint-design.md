# 新闻去重优化：关键特征指纹方案

## 背景

当前去重基于 `SequenceMatcher`（阈值 0.4），对同一事件不同措辞的报道容易漏检。例如财联社报道和公司公告引用相同数据但文字结构不同，相似度可能刚好在阈值边界。

## 方案概述

在现有 SequenceMatcher 基础上，新增**关键特征指纹**通道。从新闻文本中提取结构化特征（公司名、金额、日期），用 Jaccard 相似度判断是否同一事件。两个通道任一命中即判重复。

## 详细设计

### 1. 文本预处理

去除新闻源前缀噪音，减少干扰：

```python
PREFIXES = [
    r'^(财联社|每经|证券时报|新华社|央视)\d{1,2}月\d{1,2}日(电|讯|消息)[，,\s]*',
    r'^据.{2,10}(报道|消息|透露)[，,\s]*',
    r'^【[^】]+】\s*',
]
```

### 2. 特征提取

从清洗后文本中提取三类特征，组成特征集合（set）：

**实体名匹配**：
- 来源：`CompanyKeyword` 表（活跃条目）+ `Stock` 表 `name` 字段
- 过滤：名称长度 >= 2 字符，排除过短名称的误匹配
- 方法：将所有实体名按长度降序排列，合并为单个正则 `(三星电子|贵州茅台|...)` 编译匹配，一次扫描完成
- 懒加载：首次调用 `filter_duplicates` 时从 DB 加载（此时必定在 Flask app context 内），缓存在内存中，每小时自动刷新

**数字+单位提取**：
```python
# 金额：数字 + 量级 + 货币单位
r'\d+\.?\d*\s*[百千万亿]+[美韩日欧]?[元圆币]'
# 美元格式
r'\$[\d.]+[BMK]?'
```
保留原始文本形式不做数值换算（同一事件的不同报道通常引用相同量级表述）。
不提取百分比和股价，仅提取绝对金额。

**日期/时间引用提取**：
```python
# 年份
r'\d{4}年'
# 季度
r'[一二三四1-4]季[度报]|Q[1-4]'
# 半年/年报
r'[上下]半年|半年报|年报'
```

提取结果示例：
```
{三星电子, 110万亿韩元, 732亿美元, 2026年}
```

### 3. 双通道判定

```
is_duplicate = SequenceMatcher相似度 >= 0.4（原有）
              OR (特征数 >= 3 AND 交集数 >= 2 AND Jaccard >= 0.5)（新增）
```

指纹通道的三重条件：
- **特征数 >= 3**：两条新闻的特征集合并集大小至少为 3，避免特征过少时 Jaccard 粒度太粗
- **交集数 >= 2**：至少有 2 个共同特征，避免仅因共享一个年份就误判
- **Jaccard >= 0.5**：共享特征占总特征一半以上

当任一新闻的指纹集合为空或大小 < 2 时，跳过指纹通道，仅用 SequenceMatcher。

### 4. 推送缓冲区扩展

`_pushed_buffer` 结构从 `tuple` 改为 `NamedTuple`：

```python
class PushedRecord(NamedTuple):
    timestamp: datetime
    content: str
    fingerprint: frozenset
```

使用 NamedTuple 避免裸 tuple 的解包风险。

### 5. 跨路径统一

`filter_duplicates` 内部统一对 `content_key` 提取的内容做清洗+指纹提取。兴趣推送（`interest_pipeline.py`）和公司新闻推送（`company_news_service.py`）无需改动调用方式。

### 6. 实体名缓存

- **加载时机**：懒加载，首次调用 `filter_duplicates` 时从 DB 加载（避免模块导入时无 app context 的问题）
- **刷新策略**：TTL 1 小时自动刷新，无需手动调用
- **实现**：编译为单个正则 `re.compile('|'.join(sorted(names, key=len, reverse=True)))`

### 7. 日志增强

```
[去重] 指纹命中: {三星电子,110万亿韩元,2026年} ∩ {三星电子,110万亿韩元,2026年,9.8万亿韩元} Jaccard=0.71
[去重] 输入 5 条，过滤 2 条（文本相似1条，指纹匹配1条），推送 3 条
```

DEBUG 级别输出完整指纹集合，INFO 级别仅输出统计。

## 文件变更范围

- `app/services/news_dedup.py` — 主要改动：新增文本清洗、特征提取、指纹比较逻辑

## 已知限制

- `1.1万亿` 和 `11000亿` 不会被识别为相同金额（同一事件通常使用相同量级表述，可接受）
- 实体名匹配依赖 DB 中已有的公司/股票数据，未录入的公司名无法提取

## 不做的事

- 不引入外部 NLP 库或 embedding 模型
- 不增加额外 LLM API 调用
- 不改变调用方的接口
- 不做推送历史持久化
