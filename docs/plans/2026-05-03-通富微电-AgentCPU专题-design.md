## 0. 设计稿目的

回答用户原始问题——**通富微电是否受益于 agent 时代 CPU:GPU 配比从 1:8 变成 1:2**。

5-2 专题已经穷尽分析了 TFAMD 的 17.5% markup 框架 + AMD 周期暴露 + 槟城产能节奏，但**完全没有把"CPU:GPU 配比"作为独立变量分析过**——5-2 默认 AMD 数据中心增长按"GPU 主导"传导，未拆 CPU 维度。本专题填这个盲点。

## 1. 核心论证骨架

### 1.1 一句话设计目标

**确认/证伪"agent 时代 CPU:GPU 配比 1:2 → 通富 EPS 显著上修"这个市场叙事**——通过 4 场景独立打分 + 条件树主路径，给 5-2 评级一个"是否需要修正"的答案。

### 1.2 4 场景分类（决定整篇分析的轴）

| 编号 | 场景 | 1:2 数字怎么来 | 通富受益路径 | 预设权重 |
|------|------|------------|------------|--------|
| **S1** | NVIDIA GB200 NVL72 物理架构 | 1 Grace + 2 Blackwell（superchip 设计）；NVL72 = 36 Grace + 72 Blackwell | **❌ 零受益 + 反向威胁**（NVIDIA 抢 AMD 数据中心份额）| 中 |
| **S2** | Agent inference 工作负载 | tool-call/RAG/orchestration/KV 管理让 host CPU 占用从 5% → 30%，传导到 EPYC 出货量乘数 | **✅ 强受益**（TFAMD 同时封 EPYC + MI）| **高（核心叙事承载）** |
| **S3** | AMD MI300A APU 架构 | CPU+GPU 同 die（Zen4 + CDNA3）= 物理 1:1 极端；TFAMD 苏州独家封 | **✅ 直接但盘子小**（HPC 利基，El Capitan 量级）| 低 |
| **S4** | AMD Helios/MI400 平台默认配比上调 | AMD 改 2 EPYC + 8 MI（1:4）→ 4 EPYC + 8 MI（1:2）对标 GB200 | **✅ 强但未公开**（依赖 26-11 投资者日披露）| 中 |

## 2. 章节设计

### §0 执行摘要（~400 字）

- 一句话结论（最终落定路径）
- 4 场景 reality check 速览表
- 5-2 评级修正方向预判（升/降/不变 + 量级）
- 与 5-2 专题的明确分工：5-2 = TFAMD 资本结构 + 17.5% markup 天花板；本专题 = agent 工作负载维度对 5-2 评级的灵敏度

### §1 工作负载 reality check（~1200 字）

**论证目标**：1:2 这个数字到底是物理架构还是工作负载特征？传导到 CPU 物理出货量的转换效率是多少？

**5 个子问题（按论证顺序）**：

1. **Q1.1 — 1:2 出处溯源**：检索关键词 `CPU GPU ratio agent inference 1:2` / `GB200 NVL72 architecture` / `agent inference CPU bottleneck`
   - 数据源候选：SemiAnalysis archives / Citi GTC 26 / Morgan Stanley AI infra / NVIDIA GTC 26 主题 / AMD 26Q1 财报会议 / Anthropic Eng 推理工程帖 / OpenAI Eng / vLLM 工程博客 / SGLang benchmarks
   - 输出：1:2 这个数字的最早出现点 + 主流引用链

2. **Q1.2 — 物理 vs 工作负载二分**：
   - 物理：GB200 NVL72 是 NVIDIA 独有，与通富错配
   - 工作负载：CPU cycles 占比上升，AMD/NVIDIA 通吃
   - **关键判定**：1:2 这个具体数字是物理架构特例还是 inference workload 普遍规律？

3. **Q1.3 — agent inference 是否真的让 CPU 占用结构性倍增？**
   - 子问题：tool calling / RAG / multi-turn orchestration / KV cache 管理 / batching scheduler 哪些 CPU bound？
   - 反例视角：vLLM PagedAttention 把调度从 CPU 推回 GPU；KV cache 已下沉 GPU memory；CPU 占比是否真在升？
   - 数据源：SGLang/vLLM benchmark + NVIDIA Triton inference server 调度文档 + Anthropic 推理工程帖 + Hugging Face TGI

4. **Q1.4 — CPU 工作负载 → CPU 物理出货量的转换效率**：
   - 不是说 CPU 占用从 5% → 20% 就要 4 倍 CPU 数量；可能只是 CPU 利用率从 30% → 80%，CPU 数量不变
   - 必须区分「CPU 工作负载占比」和「CPU 物理出货量」两件事
   - 决定 EPYC 销量乘数是 1.0x（没传导）/ 1.3x（部分传导）/ 1.8x（强传导）/ 4x（CPU 数量翻倍）

5. **Q1.5 — 输出 4 档判定**：

   | 档位 | 内涵 | 对 §2-5 的影响 |
   |------|------|-------------|
   | **A 档** | 物理 1:2 + 工作负载 CPU 占用上升 = 双重利好 | S2/S4 大利好；S1 错配但不致命 |
   | **B 档** | 仅工作负载 CPU 占用上升，物理仍 1:4-1:8；EPYC 销量乘数 1.3-1.8x | S2 受益减半；S1 错配仍是威胁 |
   | **C 档** | CPU 占用上升但 CPU 销量乘数 ≤ 1.5x | S2/S4 利好打折至中性偏积极 |
   | **D 档** | 1:2 主要源自 GB200 物理架构；agent workload 没让 CPU 数量结构性放大 | S2 不成立；agent 叙事对通富 = 中性 |

**预判**：基于已读资料，§1 大概率落 **B 档**——主流引用的 1:2 大都指向 GB200 NVL72 物理架构，agent inference workload 确实让 host CPU 工作量上升但不直接带来 4x CPU 销量；EPYC 出货量乘数预计 1.3-1.8x。

### §2 S1 解析：NVIDIA GB200 NVL72 架构（~800 字）

- GB200 物理 BOM 拆解：36 Grace + 72 Blackwell + 9 NVSwitch；Grace 由 TSMC CoWoS 封装，与通富无关
- GB200 26 出货量共识：~30 万 superchip（SemiAnalysis 推算）
- AMD 数据中心 GPU 26 全年共识 $15 bn，对应 ~250 万 MI 系列 GPU；NVIDIA Blackwell 出货 ~600 万颗
- **份额冲击测算**：如果 GB200 在 26-27 抢走 AMD 数据中心 GPU 30% 份额 → AMD MI 营收预期下修 30% → TFAMD 营收下修 ~10%（GPU 占 TFAMD 营收 ~30%）→ 通富 27 EPS 下修 0.10-0.20 元
- 8 维度打分：真实性 ✅ / 概率 高 / 通富份额 0% / 营收偏移 -10 至 -20 亿 / 净利偏移 -1.0 至 -1.6 / 17.5% 天花板 不变 / 5-2 修正 降半档 / 风险 NVIDIA 客户锁定深化

### §3 S2 解析：Agent inference 工作负载（核心章，~1500 字）

- agent loop 工作流的 CPU 消耗画像：plan → tool exec → retrieve → reason 各环节的 CPU/GPU 比例
- 引述 Anthropic / OpenAI / vLLM 工程帖中 host CPU 占用率的典型数据
- 传导链测算：agent inference 在 26-27 占数据中心 inference 的比例（共识 30-50%）→ 该细分 EPYC 配比上升幅度 → AMD 全数据中心 EPYC 出货量乘数
- TFAMD 现有产能能否承接 EPYC 出货量乘数？（25Q4 槟城二厂投产 + 苏州扩产 = 26-27 增量产能 ~30%，可承接 EPYC 出货 +50% 但需要有限的 capex 追加）
- **17.5% markup 框架下净利兑现率**：
  - 营收乘数 1.3-1.8x 传导到 TFAMD 营收 ~+15-30 亿 / 27 年
  - 净利率维持 8.0-8.5%（不破天花板）→ 净利偏移 +1.2-2.4 亿
  - EPS 偏移 +0.08-0.16 元（按 14.6 亿股本）
- 8 维度打分：真实性 ⚠️（B 档）/ 概率 中 / 通富份额 80% / 营收偏移 +15-30 亿 / 净利偏移 +1.2-2.4 / 17.5% 天花板 不变 / 5-2 修正 不变 → 升半档 / 风险 EPYC 被 ARM/Grace 替代

### §4 S3 解析：AMD MI300A APU（~700 字）

- MI300A 物理设计：3 CCD（Zen4）+ 6 XCD（CDNA3）on CoWoS-S
- TFAMD 苏州独家封装（中国大陆唯一具备 MI300A 后端封装能力）
- 出货量级：El Capitan 超算 + 部分 HPC 客户，26-27 估计 ~5-10 万颗
- 营收/净利贡献：MI300A 单价高（>$10K）→ 营收 +5-10 亿 / 27 年；净利 +0.4-0.8 亿
- 8 维度打分：真实性 ✅ / 概率 中（已量产但 ramp 慢）/ 通富份额 100% / 营收偏移 +3-8 亿 / 净利偏移 +0.25-0.65 / 17.5% 天花板 不变 / 5-2 修正 不变 / 风险 AMD 战略重心从 APU 转回 discrete GPU

### §5 S4 解析：AMD Helios/MI400 平台默认配比上调（~800 字）

- Helios 26H2 公告版本：2 EPYC Venice + 8 MI400（1:4）
- 假设：AMD 26-11 投资者日改成 4 EPYC + 8 MI400（1:2）对标 GB200
- 兑现条件 + 概率（基于 AMD 历史 product roadmap 调整频率）
- 兑现后的营收偏移：EPYC 出货量乘数 +50%（AMD 数据中心 EPYC 25 → 27 路径从 $13 bn → $20 bn 升至 $13 bn → $30 bn）
- 8 维度打分：真实性 ❓（未公开）/ 概率 低 / 通富份额 80% / 营收偏移 +20-40 亿（条件兑现）/ 净利偏移 +1.6-3.2 / 17.5% 天花板 不变 / 5-2 修正 不变 → 升一档（条件触发）/ 风险 AMD 不改默认配比

### §6 综合判断：条件树（~800 字）

不做加权平均，做"如果...那么..."的三主路径：

```
主路径 A — agent 叙事真兑现（25% 概率）
├── 触发: §1 落 A/B 档 AND AMD 26-11 确认 Helios 1:2 配比
├── 营收偏移: +35 至 +65 亿
├── 净利偏移: +2.8 至 +5.2 亿
├── EPS 偏移: +0.18 至 +0.34 元
├── 27 EPS: 5-2 中性 1.85 → 2.05-2.20
├── 5-2 评级修正: 升一档（观察 → 偏积极 → 买入备选）
└── 仓位上限: 5% → 7%

主路径 B — agent 叙事局部兑现（55% 概率，最可能路径）
├── 触发: §1 落 B/C 档 AND S4 未确认 AND NVIDIA GB200 抢部分份额
├── 营收偏移: -5 至 +10 亿（接近中性）
├── 净利偏移: -0.4 至 +0.8 亿
├── EPS 偏移: -0.03 至 +0.05 元
├── 27 EPS: 5-2 中性 1.85（基本不变）
├── 5-2 评级修正: 不变（观察 — 双向风险）
└── 仓位上限: 5% 不变

主路径 C — agent 叙事证伪 + NVIDIA 强势（20% 概率）
├── 触发: §1 落 D 档 AND NVIDIA GB200 NVL72 27 抢走 AMD 数据中心 30%+ 份额
├── 营收偏移: -20 至 -40 亿
├── 净利偏移: -1.6 至 -3.2 亿
├── EPS 偏移: -0.10 至 -0.20 元
├── 27 EPS: 5-2 中性 1.85 → 1.65-1.75
├── 5-2 评级修正: 降一档（观察 → 偏谨慎）
└── 仓位上限: 5% → 3%
```
