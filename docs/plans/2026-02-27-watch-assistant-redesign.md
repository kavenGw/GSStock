# 盯盘助手优化设计

## 需求

1. 开盘时显示每60秒自动更新的实时分时线
2. 收盘/未开盘时显示上一交易日数据
3. 前端定时轮询驱动，移除后台策略扫描
4. AI分析分三个时间维度：实时、7天、30天

## 方案：重写前端 + 改造后端

### 删除的代码

- `app/strategies/watch_assistant/__init__.py` — 后台扫描策略
- `scheduler/engine.py` 中 watch_assistant 注册
- WatchAnalysis 模型的 `volatility_threshold` 字段
- 通知冷却相关逻辑

### 后端API改造

**`/watch/chart-data`**：
- 新增 `last_timestamp` 参数（可选），增量模式只返回新数据点
- 收盘时自动返回上一交易日完整分时数据

**`/watch/analyze`**：
- 新增 `period` 参数：`realtime` / `7d` / `30d`
- realtime：当日分时 + 当前价格 → 当日走势分析（不缓存）
- 7d：7天K线 → 短期趋势分析（按天缓存）
- 30d：30天K线 → 中期趋势分析（按天缓存）

**WatchAnalysis 模型**：
- 新增 `period` 字段
- 唯一约束：`(stock_code, analysis_date, period)`
- 移除 `volatility_threshold`

### 前端重写

**页面布局**：每只股票一个大卡片，内嵌实时分时线

```
┌─────────────────────────────────────────────────┐
│ 盯盘助手        [添加股票]                         │
├─────────────────────────────────────────────────┤
│ 市场状态栏                                       │
├─────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────┐ │
│ │ 股票名称 代码          价格 涨跌幅           │ │
│ │ ┌─────────────────────────────────────────┐ │ │
│ │ │         实时分时线 (ECharts)              │ │ │
│ │ └─────────────────────────────────────────┘ │ │
│ │ AI分析：[实时] [7天] [30天] tab切换          │ │
│ │ 支撑/阻力位                                 │ │
│ └─────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────┘
```

**交互逻辑**：
- 页面加载：获取列表 + 市场状态 + 价格 + 分时数据 + AI分析
- 开盘中：每60秒轮询价格和增量分时数据，追加到图表
- 收盘/未开盘：加载上一交易日完整分时数据，不轮询
- AI分析按钮：三维度并行请求，有缓存直接显示

### LLM Prompt

三个维度各自关注不同重点：
- 实时：当日走势解读、短线信号
- 7天：短期趋势、支撑阻力、量价关系
- 30天：中期形态、趋势方向、关键位置
