# 新闻影响分析 rubric + theme 档模板

本文件是 news-impact skill 的执行底料：怎么判方向/量级、怎么接旧 thesis、theme 档怎么写、related_docs 怎么对称。SKILL.md 流程到对应步骤再读这里。

## 一、传导判定 rubric（每个候选标的填一行）

对每个命中标的，**不是问"涨还是跌"，而是问"这条新闻改变了它 doc 里旧 thesis 的哪个变量"**。这是本 skill 的全部价值——脱离旧 thesis 的 sentiment 打分谁都会做，没意义。

| 维度 | 取值 | 判定要点 |
|------|------|---------|
| 传导路径 | 直接命中 / 同业(sector,subsector) / 上下游(在池内能找到) / 主题(themes 关键词) | 必须能用一句话说清"新闻里的事 → 怎么传到这只票的收入/成本/估值"，说不清就是噪音不要列 |
| 方向 | 利好 / 利空 / 中性偏多 / 中性偏空 / **无影响** | "无影响"是合法且重要的结论：硬凑关联比漏标的更伤可信度 |
| 量级 | 高 / 中 / 低 | 高=动摇核心收入或重估逻辑；中=边际影响一个分部；低=情绪扰动无基本面 |
| 时间窗 | 即时(情绪) / 季度内(订单/价格) / 中长期(产能/格局) | 区分"明天开盘的事"和"两年的事" |
| 对旧 thesis | 强化 / 动摇 / 推翻 / 无关 | 落到 doc 里旧评级——这条新闻让 core 更稳？让 watch 该升 config？让 exclude 该重看？ |
| 是否已 priced in | 是 / 否 / 存疑 | 已是市场共识的旧闻，对价格无增量；只标"新增量信息" |
| 置信度 | 高 / 中 / 低 | 取决于新闻本身是确认事实还是传闻、以及传导链条几阶 |

**纪律（照搬 stock-deep-redo 的硬规矩）**：
- **先联网核实新闻**，再分析。用户粘的文本可能是传闻/标题党/已被辟谣。URL 用 WebFetch 抓原文；关键数字交叉验证 2-3 家。新闻真伪不明时，置信度直接压到"低"并在档里写明。
- **供给侧/利好论点必须双面**：一条"扩产"既是下游成本利好也是上游价格利空；一条"涨价"利好厂商但要问需求端能否承接。不要单边叙事。
- **二阶传导留在池内**：本 skill 范围是 docs 池（不外扩产业链图谱），但池内的上下游/同业联动要追到（如某材料涨价 → 池里用该材料的下游标的）。
- **反向证据优先**：若某标的 doc 或其 IR/管理层口径明确说过"此类事件无影响"，那是强反向证据，按"无影响"标注并引用，不要为了凑数翻案。

## 二、覆盖 vs 深度：全员进表，只深写中量级以上

两个目标同时要：**不漏**（弱相关标的也要被扫到、判过、列出）+ **不灌水**（不在弱标的上堆篇幅）。做法是把"列出"和"展开"分开：

- **全员进总览表**：召回的每个候选标的都占一行，按传导强度排序，方向列允许取"弱利好""无影响"，量级列取 高/中/低。弱相关、无影响的**照样列**并标注——这是"我扫过、判过、没漏"的证据，不是灌水。
- **只深写中/高量级**：量级达到中或高的才单开逐标的分析段（双面 + 接旧 thesis + 操作含义）。低/弱/无影响的留在表里一行带过，或合并成一句"扫描后判定无/弱传导：清单 + 各一句话理由"。

召回时按这四档强度排，但**都要进表**：
1. 直接命中（新闻点名的公司/产品）
2. 同 subsector 直接竞品 / 上下游池内标的
3. 同 sector 但隔了 subsector
4. 仅 themes 关键词擦边

同一标的多篇 doc 时，以最新 `conviction_date` 的 buffett 档为权威评级锚，其余 doc 进 related_docs。

## 二之二、建档门槛

总览表填完后看最高量级：**至少一只达到"中"或以上 → 建 theme 档**；**全是低/弱/情绪级 → 默认不建档**，只在对话给表+结论并问用户是否仍要归档为主题背书。理由：档案进池会成为后续选股/复查的信噪源，给弱传导事件造档是污染池子。相关性快筛已早退的"纯无交集"事件根本不到这一步。

## 三、theme 档模板

落档到 `docs/stock-analytics/themes/YYYY-MM-DD-<主题>.md`（日期=今天）。frontmatter 字段集见 `scripts/_docs_schema.py` 的 `theme` 类型，**必填**：`doc_type` `theme_name` `themes` `date`。

```markdown
---
doc_type: theme
theme_name: <主题简称，如 "美光 DDR4 扩产影响分析">
themes:
  - <关键词1>
  - <关键词2>
related_codes:
  - '603986'        # 受影响标的代码，字符串引号防 YAML int 化丢前导 0
  - '300223'
date: '2026-05-31'  # 今天，字符串
related_docs:
  - path: ../sectors/<sector>/<subsector>/<受影响个股 buffett 档>.md
    note: <一句话说明这只票为何被这条新闻影响>
    symmetric: true
---
# <主题>专题（YYYY-MM-DD）
```

## 四、related_docs 对称收尾（必做）

theme 档在 frontmatter 里引用了受影响个股 doc（`symmetric: true`），所以那些个股 doc 也要反向引用回 theme 档，否则 `lint_docs_refs.py` 报不对称。两种做法二选一：

- **省事**：写完 theme 档后跑 `python scripts/lint_docs_refs.py --rewrite-blocks`，它按 frontmatter 重生所有顶部 markdown 块。但反向 frontmatter 条目仍需手加。
- **稳妥**：手动给每个受影响个股 doc 的 frontmatter `related_docs` 加一条指回 theme 档（`symmetric: true`），再跑 `--rewrite-blocks`。

收尾两条命令（路径用 repo 根的 scripts/，不是 skill 内）：
```bash
python scripts/lint_docs_frontmatter.py     # 新 theme 档 frontmatter 合法性
python scripts/lint_docs_refs.py            # related_docs 路径 + 反向对称
```
退出码 0 = 过。非 0 按清单补。
